# 鹦鹉管理系统开发任务拆分清单

> 本清单根据 `prompt.md` 和《基于 Spring Boot 的鹦鹉管理系统需求分析说明书》整理。开发时以“能完成、能演示、能答辩、能写论文”为核心，不追求过度复杂，但要保证登录、权限、CRUD、分页、上传、图表、前后台闭环完整。

## 一、开发通用约束

1. 所有页面文案、注释、字段说明、业务名称优先使用简体中文。
2. 代码风格要像普通学生项目，命名自然，避免过长、过整齐、过模板化。
3. 类、核心方法、关键业务逻辑、SQL 查询条件处要写中文注释，但不要每行都解释。
4. 后端接口统一返回 `code`、`message`、`data`，禁止混用 `msg`。
5. 默认账号必须由后端 `DataInitializer` 启动初始化，禁止写入 `data.sql`。
6. 前端统一使用“鹦羽青绿”主题：主色 `#16A085`，辅色 `#2ECC71`，强调色 `#F4B400`。
7. 前端界面开发必须参考 `UI/code_artifact.tsx` 的整体布局、配色、卡片、表格、标签页、前台内容页和交互风格，用 Vue3 + Element Plus 实现，不直接照搬 React 代码。
8. 每个阶段完成后都要保证能运行、能演示一个小闭环，再进入下一阶段。

# 答辩展示建议流程

1. 使用 `admin / admin123` 登录后台，展示首页看板和权限菜单。
2. 进入品种管理，新增或编辑一个品种。
3. 进入鹦鹉档案，新增一只鹦鹉并上传图片，设置为公开。
4. 切到前台，展示这只鹦鹉出现在公开列表中。
5. 使用 `customer / admin123` 登录，提交预约咨询。
6. 使用管理员或饲养员回到后台，确认并完成预约。
7. 客户查看“我的预约”，看到状态和处理结果。
8. 饲养员新增健康记录并填写异常症状，展示鹦鹉状态变为 `观察中`。
9. 管理员发布一条饲养知识，前台公告知识列表显示。
10. 展示登录日志，说明系统记录了登录成功和失败情况。
